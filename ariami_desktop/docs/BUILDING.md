# Building Ariami Desktop from Source

This supplements the "Building" section in the existing
`ariami_desktop/README.md` with platform-specific prerequisites verified
against the actual project files.

## Common prerequisites

From the repository root `GUIDE.md`:

- **Dart SDK 3.5.0+** (`ariami_desktop/pubspec.yaml`: `environment: sdk: ^3.5.0`)
- **Flutter SDK**, latest stable
- The sibling `ariami_core` package checked out at `../ariami_core` relative
  to `ariami_desktop` — the dependency is a relative path, not a pub.dev
  package (`ariami_desktop/pubspec.yaml`: `ariami_core: { path: ../ariami_core }`)

```bash
cd ariami_desktop
flutter pub get
```

## Sonic transcoder (optional native dependency)

Low/medium-quality server-side transcoding is powered by a bundled Rust
library ("Sonic") built from the `sonic/` directory at the repository root —
a **git submodule**. It is referenced by relative path from both the Linux
and (implicitly, via a prebuilt framework) macOS build outputs:

```cmake
# ariami_desktop/linux/CMakeLists.txt
set(SONIC_DIR "${CMAKE_CURRENT_SOURCE_DIR}/../../sonic")
...
if(EXISTS "${SONIC_DIR}/Cargo.toml")
  add_custom_target(build_sonic_library ALL
    COMMAND ${CMAKE_COMMAND} -E env CARGO_TARGET_DIR=${SONIC_TARGET_DIR} cargo build --release --features aac-fdk --lib
    WORKING_DIRECTORY "${SONIC_DIR}" ...)
else()
  message(WARNING "Sonic source not found at ${SONIC_DIR}; low/medium transcoding will be disabled.")
endif()
```

If you cloned the repository without submodules, fetch it first:

```bash
git submodule update --init --recursive
```

Building it also requires a working **Rust + Cargo** toolchain on the machine
running the Linux build (the `cargo build --release --features aac-fdk --lib`
invocation above runs as part of the normal CMake/Flutter build — you do not
invoke Cargo yourself). If the submodule is missing, the Linux build still
succeeds; it just skips bundling the library and transcoding falls back to
being disabled (see `docs/TROUBLESHOOTING.md`).

## macOS

- **Deployment target:** macOS 10.15 (`MACOSX_DEPLOYMENT_TARGET = 10.15` in
  `ariami_desktop/macos/Runner.xcodeproj/project.pbxproj`).
- **Bundle identifier:** `com.example.ariamiDesktop`
  (`ariami_desktop/macos/Runner/Configs/AppInfo.xcconfig`).
- **Sandbox:** disabled (`com.apple.security.app-sandbox` is `false` in both
  `ariami_desktop/macos/Runner/Release.entitlements` and
  `DebugProfile.entitlements`).
- Xcode + the Flutter macOS desktop toolchain must be installed and accepted
  (`sudo xcodebuild -license` if you've never opened Xcode).

```bash
cd ariami_desktop
flutter run -d macos      # debug run
flutter build macos       # release build; produces a .app under build/macos/Build/Products/Release
```

The app icon is generated via `flutter_launcher_icons`
(`ariami_desktop/pubspec.yaml`, `flutter_launcher_icons: macos:` section),
sourced from `assets/Ariami_icon_macos.png`, pre-padded to Apple's icon grid
per the comment in `pubspec.yaml`.

## Windows

- Uses the standard Flutter Windows desktop toolchain (CMake + a
  Visual Studio C++ toolchain), per
  `ariami_desktop/windows/CMakeLists.txt` (`cmake_minimum_required(VERSION 3.14)`).
  Enable the "Desktop development with C++" workload in Visual Studio if
  `flutter doctor` flags it as missing.
- Binary/product name: `ariami_desktop`
  (`set(BINARY_NAME "ariami_desktop")` in `windows/CMakeLists.txt`).
- Version info baked into the executable comes from
  `ariami_desktop/windows/runner/Runner.rc`, driven by `FLUTTER_VERSION_*`
  build defines (falls back to the literal string `"4.2.0"` in the `.rc`
  file if those defines aren't present at all, which is a Flutter template
  fallback and not the actual shipped version — the real version always
  comes from `pubspec.yaml`'s `version:` field via those build defines).

```bash
cd ariami_desktop
flutter run -d windows
flutter build windows     # produces build\windows\x64\runner\Release\
```

## Linux

- **Requires GTK 3 development headers** at build time
  (`ariami_desktop/linux/CMakeLists.txt`:
  `pkg_check_modules(GTK REQUIRED IMPORTED_TARGET gtk+-3.0)`), for example on
  Debian/Ubuntu:

  ```bash
  sudo apt update
  sudo apt install clang cmake ninja-build pkg-config libgtk-3-dev
  ```

  (`clang`/`cmake`/`ninja-build` are Flutter's own general Linux desktop
  build prerequisites; `pkg-config` and `libgtk-3-dev` are what the
  `gtk+-3.0` check above needs.)
- **Rust + Cargo** if you want the Sonic transcoder built (see above); the
  build proceeds without it, just with transcoding disabled.
- Project/binary name: `ariami_desktop`
  (`ariami_desktop/linux/CMakeLists.txt`:
  `set(BINARY_NAME "ariami_desktop")`; `APPLICATION_ID` defaults to the
  Flutter template value `com.example.ariami_desktop`).
- The Linux window icon is **not** generated by `flutter_launcher_icons`
  (which has no Linux support, per the comment in
  `ariami_desktop/pubspec.yaml`) — it must be replaced by hand at
  `ariami_desktop/linux/runner/resources/app_icon.png`.

```bash
cd ariami_desktop
flutter run -d linux
flutter build linux       # produces build/linux/x64/release/bundle/
```

## Running tests

`ariami_desktop/test/` contains widget and service-level unit tests, run the
usual way:

```bash
cd ariami_desktop
flutter test
```

Existing tests cover, among other things, the desktop download-limits
service, transcode-slots service, Spotify import service, update-check
service, onboarding help copy, and the connected-clients/user-activity table
widgets (see file names under `ariami_desktop/test/`).
