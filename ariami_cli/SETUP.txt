ARIAMI CLI - RASPBERRY PI SETUP
================================

REQUIREMENTS
------------
- Raspberry Pi 3, 4, or 5 (ARM64)
- Raspberry Pi OS (64-bit)
- unzip utility (usually pre-installed)

INSTALLATION
------------
1. Extract the archive:
   unzip ariami-cli-raspberry-pi-arm64-v4.4.0.zip
   cd ariami-cli-raspberry-pi-arm64-v4.4.0

2. Make the binary executable:
   chmod +x ariami_cli

3. Start the server:
   ./ariami_cli start

FIRST RUN
---------
On first run, the server will:
- Ask whether Ariami should start automatically on boot (y/N)
- Start in foreground mode
- Open a browser for setup once the server is ready
- Walk you through Tailscale (optional), music folder selection, and library scan
- Prompt you to create the owner account (first account = server admin)
- Show a QR code so phones can register or log in

Complete these steps in the browser before setup is finished.

If the browser does not open automatically, navigate to:
http://localhost:8080

USAGE
-----
Start server (background):
  ./ariami_cli start

Check status:
  ./ariami_cli status

Stop server:
  ./ariami_cli stop

Custom port:
  ./ariami_cli start --port 8081

START ON BOOT
-------------
Make the server start automatically after a reboot/power loss:
  ./ariami_cli autostart enable

Disable it again:
  ./ariami_cli autostart disable

Check the current setting:
  ./ariami_cli autostart status

On first-time setup you are also asked this as a y/N question. Use the
commands above to change it later, or to enable it on an install that was
set up before this option existed. On Raspberry Pi this uses an "@reboot"
crontab entry for the current user (no sudo required).

RESET
-----
Start over without deleting your music files:
  ./ariami_cli reset

You will be asked to choose:
  1. Setup/config only   (keeps your library database and accounts)
  2. Factory reset Ariami data   (removes database, accounts, sessions, cache)
  3. Cancel

Then type RESET to confirm. If the server is running it is stopped first.
A factory reset also disables start-on-boot. Your music folder is never
deleted. Non-interactive forms are available for scripts:
  ./ariami_cli reset --setup -y
  ./ariami_cli reset --factory -y

CONFIGURATION
-------------
Configuration files are stored in:
  ~/.ariami_cli/

This includes:
- Music folder path
- Server settings
- Process ID file

TRANSCODING
-----------
The release package includes:
- lib/libsonic_transcoder.so

With this file present, low/medium quality requests transcode correctly.
If low/medium requests fall back to high quality, verify:
- The file exists at: ./lib/libsonic_transcoder.so
- You started the server from the extracted release directory

MOBILE APP
----------
After owner setup, scan the QR code in the web interface with
the Ariami mobile app, then register a new account or log in.
Registration uses a time-limited token embedded in the QR code.

TROUBLESHOOTING
---------------
Server not responding:
- Check if running: ./ariami_cli status
- Check firewall allows port 8080 (or the port shown by status)
- Check logs in foreground: ./ariami_cli start (after stopping)

Port 8080 already in use:
- Ariami tries 8081–8099 automatically on first start
- Run ./ariami_cli status to see the port in use
- Or pin a port: ./ariami_cli start --port 9000

Reset configuration:
- Stop server: ./ariami_cli stop
- Remove config: rm -rf ~/.ariami_cli/
- Restart server: ./ariami_cli start

TAILSCALE (OPTIONAL)
--------------------
For remote access without port forwarding, install Tailscale:
https://tailscale.com/download/linux/rpi

The server will detect and use Tailscale if installed.
